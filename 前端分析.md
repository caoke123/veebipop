## 状态管理模式

- 专用状态库结论
  - 未检测到 `redux`/`zustand` 等依赖；项目未使用这类第三方状态管理库。
- React Context API（全局/半全局状态）
  - 上下文目录：`src/context`
    - 文件：`CartContext.tsx`, `WishlistContext.tsx`, `CompareContext.tsx`
    - 模态相关：`ModalCartContext.tsx`, `ModalCompareContext.tsx`, `ModalQuickviewContext.tsx`, `ModalSearchContext.tsx`, `ModalWishlistContext.tsx`
  - 典型实现片段（以购物车为例 `src/context/CartContext.tsx`）：
    ```
    const CartContext = createContext<CartContextProps | undefined>(undefined);

    export const CartProvider: React.FC<{ children: React.ReactNode }> = ({ children }) => {
      const [cartState, dispatch] = useReducer(cartReducer, { cartArray: [] });
      // addToCart/removeFromCart/updateCart...
      return (
        <CartContext.Provider value={{ cartState, addToCart, removeFromCart, updateCart }}>
          {children}
        </CartContext.Provider>
      )
    }
    ```
  - 全局注入位置
    - `src/app/GlobalProvider.tsx`：聚合并包裹所有 Context Provider
    - `src/app/ClientRoot.tsx`：在根布局中使用 `GlobalProvider` 包裹页面，并挂载各模态组件
    - `src/app/layout.tsx`：根布局调用 `ClientRoot`
- 轻量“store”与自定义 Hook（非第三方库）
  - 目录：`src/store`
    - 文件示例：`countdownTime.tsx`（纯计算方法）、`useWishlist.tsx` 等
    - `useWishlist.tsx` 片段：
      ```
      const [wishlist, setWishlist] = useState<WishlistItem[]>([]);
      const addToWishlist = (product: Array<ProductType>) => { ... }
      const removeFromWishlist = (productId: string) => { ... }
      ```
  - 结论：此目录主要为工具函数与局部状态 Hook，并非集中式状态管理框架。


以上扫描结果为项目的当前“前端数据与状态形态”基线。结合 WooCommerce REST API 的后端对接，建议从现有的 `fetch('/Product.json')` 使用点入手，逐步替换为对 WooCommerce 的产品/分类等端点的调用，同时保持 Context Provider 的状态更新逻辑不变（或按需扩展）。如需我继续给出对接策略与最小改造方案，我可以基于上述页面与数据源清单进行详细规划。
